# OpenLeads v4.0 — design spec

**Date:** 2026-06-12
**Status:** approved (autonomous build + ship)
**Prior art:** v2 (engine+sources), v3 (deliverability tiers), v3.5 (scheduler/assistant/drip)

## Problem

v3.5 has a healthy foundation (stdlib-only engine, 247 green tests, clean source
registry, strong email-deliverability core) but the **lead generation itself is
weak**: only the YC source reliably produces emailable people. NPI is domain-less,
GitHub/OpenAlex filter to near-zero, ProductHunt/HN are company-only discovery, and
`domains` requires you to already know the domain. Worse, the engine resolves a
query through **exactly one source**, so the user must guess the right one — unlike
Apollo, which is one box that "just works." Automation exists but is not cohesive;
the chat has refinement bugs; the dashboard is a thin SPA.

## Thesis — a federated waterfall engine

Replicate how Apollo/Clay/Hunter actually work, free + local, by **federating
authoritative public datasets** instead of one proprietary DB:

1. **Company discovery** — fan out across free, keyless sources matched to the
   query's shape (geo/category/role), not one picked source.
2. **People discovery** — for each company domain, find decision-makers from public
   team/about/leadership pages.
3. **Email waterfall** — ground-truth harvest → learned pattern → permutation →
   MX/SMTP/Gravatar consensus → calibrated 0–100 confidence (already strong; reused).
4. **Dedupe + rank** — merge across sources, dedupe by domain+person, rank by
   confidence; return a verified, sendable list.
5. **Sequence + automate** — recipes, reply-aware cadences, exports, webhooks.

The headline new capability is **local-business lead-gen via OpenStreetMap /
Overpass** (category + city, global, keyless) — the long-tail that paid tools
charge most for. Plus **team-page people discovery** and **bring-your-own-list
enrichment** (Clay-style).

## Architecture

```
openleads/
  sources/
    local.py        NEW  OSM/Overpass — businesses by category + place
    companies.py    NEW  Wikidata — companies by industry + country (SPARQL)
    edgar.py        NEW  SEC EDGAR — US public companies + named officers
    yc/hn/github/openalex/npi/producthunt/domains   (existing; npi+others upgraded)
  discover/
    people.py       NEW  team/about/leadership page → [{name,title}] per domain
    geo.py          NEW  place-name → Overpass area / bbox helpers
  federation.py     NEW  fan-out planner: query → set of (source, subquery); merge/dedupe/rank
  enrich.py         NEW  BYO CSV/list of {name,company|domain} → verified emails
  engine.py         upgraded: build_leads() gains a federated path
  automate/
    recipes.py      NEW  named ICP+message+schedule; list/run/save/delete
    exporters.py    NEW  CSV/JSON/Sheets(CSV)/Notion/Airtable/webhook sinks
    sequences.py    upgraded: reply/bounce auto-pause
    scheduler.py    upgraded: recipes + watchers in the daily tick
    watch.py        NEW  alerts: "notify when new companies match my ICP"
  web/static/       rebuilt SPA: Search·Results·Lists·Sequences·Automations·Analytics·Settings
  cli.py            overhauled: `init` wizard, `enrich`, `recipe`, `export`, JSON-everywhere
  chat.py           fixed refinement + federated search
```

### Federation planner (`federation.py`)

`plan(query) -> list[SourcePlan]` maps a parsed `Query` to the sources worth
running, with per-source subqueries:

- A **place + business category** ("agencies in Miami", "dentists in Austin") →
  `local` (Overpass). This is the new default for local/SMB intent.
- **startup/founder/tech** terms → `yc` + `hn`.
- **company/industry/country** with no person role → `companies` (Wikidata) + `edgar`.
- **developer/researcher/doctor** → `github` / `openalex` / `npi`.
- A **named domain** → `domains`.
- Always honor an explicit `--source`/`/source` pin (skip federation).

`run(plans) -> Iterator[Entity]` interleaves source generators round-robin so the
engine sees a diverse stream early; the engine's existing concurrent email
resolution and early-stop are reused unchanged. `dedupe(entities)` collapses by
`(domain, normalized_name)` keeping the richest record.

### People discovery (`discover/people.py`)

`find_people(domain) -> list[{name,title,profile}]` fetches `/`, `/team`,
`/about`, `/about-us`, `/leadership`, `/our-team`, `/people`, `/company/team`
(concurrent, short timeout, cached), then extracts person blocks via:
- JSON-LD `Person` objects,
- heading/figure patterns ("Name — Title" / "Name, Title"),
- conservative name+role heuristics (reuses `_is_person_name`).
Entities are produced with `domain` set so the existing email waterfall resolves
them. Pure parsing (`extract_people(html)`) is unit-tested offline.

### New sources

- **local (Overpass):** POST/GET an Overpass QL query for nodes/ways tagged with a
  category (office=, shop=, amenity=, healthcare=, craft=) within an area resolved
  from the place name (Nominatim → area id / bbox, cached). Yields businesses with
  `website`/`contact:email` → domain → waterfall. Honest skip when no website.
- **companies (Wikidata):** SPARQL `?company wdt:P452 industry; wdt:P17 country;
  wdt:P856 website` → domain. Keyless, polite UA.
- **edgar (SEC):** company tickers JSON + submissions → company + officer names
  (from filings) + domain guess from company website where present.

### Enrichment (`enrich.py`)

`enrich_rows(rows) -> list[Lead]` accepts dicts with any of `name/first/last`,
`company`, `domain`, `email`. If `email` present → verify it; else resolve via the
waterfall using `domain` (or derive domain from `company` website lookup). Powers
`openleads enrich leads.csv` and the dashboard "Enrich a list" panel.

### Automation

- **recipes.py:** a recipe = `{name, query, count, context, schedule, verified_only,
  export, sequence}` persisted in the DB. `run(recipe)` does find→write→send/export.
  The daily tick runs due recipes (replaces ad-hoc "scheduled campaigns").
- **exporters.py:** pluggable sinks. CSV/JSON local; "Sheets" = CSV import file +
  instructions; Notion/Airtable via their REST APIs when a token is configured;
  `webhook` POSTs NDJSON of new leads to a URL. All optional, all keyless-by-default.
- **sequences.py:** before sending a follow-up, check CRM status — if `replied` or
  `bounced`, skip + mark the sequence paused. `inbox.scan` feeds status.
- **watch.py:** a watcher = a saved query + a sink; the tick diffs against
  previously-seen domains and emits only new matches.
- **suppression:** unchanged DB suppression list; federation + enrich consult it so
  no address is surfaced/sent twice.

### Interfaces

- **Web dashboard rebuild** (stdlib server unchanged; new static SPA): top search
  bar drives the federated engine with live NDJSON streaming; tabs for Results,
  Lists (saved segments + CRM), Sequences, Automations (recipes + watchers +
  schedule), Analytics (sent/reply/bounce per recipe), Settings (mailbox, tokens,
  warmup). New API endpoints: `/api/enrich`, `/api/recipes`, `/api/export`,
  `/api/analytics`, `/api/lists`.
- **CLI overhaul:** add `init` (interactive onboarding), `enrich`, `recipe`
  (list/add/run/rm), `export`; `--json` on read commands; keep all existing verbs
  working (back-compat).
- **Chat fixes:** correct the "only verified" refinement (tier-based now), route
  free text through the federated engine, add `/recipe` and `/enrich`.

## Constraints (unchanged, non-negotiable)

- **Zero hard dependencies** — stdlib only for the engine/server. New sinks
  (Notion/Airtable) use stdlib `urllib`. `rich`/`prompt_toolkit` stay optional.
- **Keyless by default** — every new source works with no API key. Tokens only
  *raise limits* or enable optional sinks.
- **Local-first** — nothing leaves the machine except the public-dataset fetches the
  engine already makes (and user-configured sinks/sends).
- **No secrets/PII in git** — leads CSVs and `.env` stay untracked.
- **Back-compat** — CSV schema only appends columns; existing CLI verbs unchanged;
  247 existing tests stay green.

## Testing

Every pure parser is unit-tested offline with fixture payloads (Overpass JSON,
Wikidata SPARQL JSON, EDGAR JSON, team-page HTML). Federation planning, dedupe,
enrichment, recipes, exporters, and sequence reply-pause get dedicated tests.
Network calls are never made in tests (sources take injected fixtures or are
monkeypatched). Target: all prior tests green + ~60 new tests.

## Rollout

Single `v4.0.0` release on the existing tag-driven workflow (GitHub release +
PyPI + npm + Pages). Version bumped in `openleads/__init__.py`, `npm/package.json`,
`pyproject` reads dynamic/static as today. CHANGELOG + README + docs + marketing
site updated to reflect the federated engine and new capabilities.

## Out of scope (explicitly)

Paid data providers, headless-browser scraping (fragile/ToS), LinkedIn scraping,
any feature that requires a mandatory API key, and any cloud component. The product
stays free, keyless, and local.
