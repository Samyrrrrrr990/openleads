the part that actually matters

      
A deliverability engine that doesn't rely on port 25.

      
Most free finders verify with a single SMTP RCPT probe over outbound
        port 25 — which home ISPs and clouds block. When it's blocked, every address quietly
        degrades to a first.last@ guess, and your campaign bounces.

      
OpenLeads cross-checks seven independent signals — most needing no port 25
        at all — then gates honestly into three tiers so you only send what's likely to land.

Nothing leaves your laptop.

    
No hosted backend. No accounts. No tracking pixels. Your leads, your drafts, your mailbox
       credentials, and your CRM all live in a local SQLite file under ~/.openleads.
       The engine talks only to public data sources and your mail server.

  







  

    

      
try it in 10 seconds

      
Install & go.

    

    

      

        
Python (recommended)

        
# zero-dependency engine + the full local app
pip install "openleads[all]"

# launch the local console (the 4 clicks, in your browser)
openleads web

# or one-shot the whole pipeline from the terminal
openleads run "50 fintech founders, verified only"

      

      

        
Node / npx

        
# no install — wraps the Python CLI
npx openleads find "rust devs in Berlin"

# interactive chat REPL — just type what you want
npx openleads

# add your own vertical
cp lawyers.py ~/.openleads/sources/
